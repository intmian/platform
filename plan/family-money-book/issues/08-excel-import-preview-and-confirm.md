# Issue 08: Excel 历史导入预览和确认

## 阶段

后续迁移能力

## 目标

管理员可以导入历史 `家庭账户.xlsx`，先预览识别结果和异常，再确认创建历史确认批次，用于初始化看板趋势。

## 范围

1. 后端接口：
   - `POST /misc/money/import/excel/preview`
   - `POST /misc/money/import/excel/confirm`
2. 前端路由：
   - `/money/:bookId/import`
3. 预览阶段识别：
   - 日期 sheet
   - 现金汇总区域
   - 资产投资区域
   - 债务区域
   - 债权区域
   - 汇总区
   - 大事记
4. 确认阶段创建 `confirmed` 历史批次。
5. 重复导入同一个 sheet 时避免重复创建批次。
6. 导入批次记录：
   - `source=excel_import`
   - 原 sheet 名

## 识别原则

1. sheet 名形如 `26-04-06`、`25-12-31`、`25-11-2` 时解析为对账日期。
2. `A:D` 附近的 `现金-汇总` 区域解析为家庭账户账面、实际、平账差额。
3. `资产-投资` 区域解析为项目上期值、当期值、当期变化、年化变化率。
4. `债务` 区域解析为负债项目和账面值。
5. `债权` 区域解析为债权项目和账面值。
6. `F:J` 汇总区解析为现金、净资产、负债、负债率、总资产和净资产变化。
7. `L:P` 上一期值粘贴区只作为校验辅助，不作为新系统数据来源。

## 非目标

1. 不把 Excel 作为后续日常记账方式。
2. 不静默丢弃无法识别的数据。
3. 不在上传后直接落库。
4. 不要求第一版支持任意 Excel 模板。

## 验收标准

1. 上传后先生成预览，不创建批次。
2. 预览列出将创建的批次、项目映射、异常 sheet、异常项目和公式风险。
3. 管理员确认后才创建历史批次。
4. 重复确认同一批 sheet 不重复创建批次。
5. 导入后的历史批次能被看板趋势读取。
6. 导入后系统重算值与 Excel 公式值不一致时，预览中明确提示差异。
