# Issue 03: 对账批次录入

## 阶段

第一部分：账目管理和对账

## 目标

管理员可以创建对账批次，从上一确认批次带入上期值，并录入本期账面值、实际值、资产负债值、间隔天数和大事记。

## 范围

1. 后端接口：
   - `POST /misc/money/batch/create`
   - `POST /misc/money/batch/get`
   - `POST /misc/money/batch/update`
2. 前端路由：
   - `/money/:bookId/reconcile/:batchId`
3. 创建新批次时：
   - 默认日期为当前日期。
   - 默认状态为 `draft`。
   - 从上一确认批次带入每个项目的上期值。
4. 批次录入内容：
   - 平账账户账面值。
   - 平账账户实际值。
   - 投资、外汇、资产、债务、债权的当期值。
   - 记账间隔天数。
   - 大事记。
5. 草稿批次允许反复保存。

## 非目标

1. 不确认批次。
2. 不展示最终看板。
3. 不做 Excel 导入。

## 页面要求

1. 对账录入区域和平账建议区域分开。
2. 平账账户需要同时录入账面值和实际值。
3. 非平账资产类项目至少录入当期值。
4. 金额输入需要避免非法字符导致后端计算异常。

## 验收标准

1. 管理员可以从账本进入新建批次。
2. 有上一确认批次时，新草稿自动带入上期值。
3. 无上一确认批次时，新草稿允许从空值开始录入。
4. 草稿保存后刷新页面仍能恢复录入内容。
5. 已禁用项目不参与新批次默认录入。
6. 前端完成新建、编辑、保存草稿的交互验证。
