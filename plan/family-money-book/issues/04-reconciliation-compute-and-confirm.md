# Issue 04: 对账计算和确认

## 阶段

第一部分：账目管理和对账

## 目标

系统根据批次录入数据计算平账建议、投资盈利和资产负债汇总，并允许管理员确认批次。

## 范围

1. 后端接口：
   - `POST /misc/money/batch/compute`
   - `POST /misc/money/batch/confirm`
2. 前端在对账页面展示：
   - 平账建议
   - 投资盈利计算
   - 资产负债汇总
   - 计算明细
3. 确认批次时保存计算结果快照。

## 平账建议规则

1. 非主账户差额 = 实际值 - 账面值。
2. 债务账户参与计算时按负数处理。
3. 非主账户差额大于 0：主平账账户转入该账户。
4. 非主账户差额小于 0：该账户转入主平账账户。
5. 根据所有非主账户差额推导主平账账户理论余额。
6. 主账户实际值 - 主账户理论余额小于 0：未知去向支出。
7. 主账户实际值 - 主账户理论余额大于 0：未知来源收入。

## 投资盈利规则

1. 当期变化 = 当期值 - 上期值。
2. 年化变化率 = 当期变化 / 当期值 * 365 / 记账间隔天数。
3. 投资盈利汇总 = 所有 `includeInInvestmentProfit=true` 项目的当期变化之和。

## 资产负债规则

1. 现金 = 所有 `includeInCash=true` 项目的当期值之和。
2. 负债 = 所有 `includeInLiability=true` 项目的当期值之和。
3. 净资产 = 参与净资产统计的正资产之和 - 负债。
4. 总资产 = 净资产 + 负债。
5. 净资产变化 = 本期净资产 - 上一期净资产。
6. 净资产负债率 = 负债 / 净资产。
7. 资产负债率 = 负债 / 总资产。

## 非目标

1. 不自动写入随手记。
2. 不把平账建议和投资盈利混为一类。
3. 不允许普通用户查看对账过程和平账建议。

## 验收标准

1. 后端计算结果可重复，保存前后结果一致。
2. 平账建议逐条说明来源账户、账面值、实际值和差额。
3. 投资盈利与平账建议独立展示。
4. 资产负债汇总展示计算口径或明细。
5. 管理员确认后批次状态变为 `confirmed`。
6. 已确认批次记录 `confirmedBy` 和 `confirmedAt`。
7. 单元测试覆盖主账户差额、债务账户负数处理、投资盈利、负债率计算。
